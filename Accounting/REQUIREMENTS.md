# 要件一覧 — Accounting(会計経理支援システム)

モノレポ全体の要件はルート [../REQUIREMENTS.md](../REQUIREMENTS.md)。本表は Accounting 固有。
優先度: **必須** / **推奨** / **任意**。状態: **実装済**(実装+テスト) / **Phase N**(将来) / **仕様**(定義のみ)。

## AC. 基盤・足場

| ID | 要件 | 優先度 | 状態 |
|---|---|---|---|
| REQ-AC01 | ADR-0011 準拠の疎結合プロジェクトとして `Accounting/` を構成する(自前 CLAUDE.md・docs・管理表・workspace) | 必須 | 実装済 |
| REQ-AC02 | `core/` は `Magi`/`TradeCouncil` を import しない(削除可能性。`shared/` 依存は scripts/scenarios のみ) | 必須 | 実装済 |
| REQ-AC03 | 会計ポリシー・会社特有論点・注意点・運用マニュアルを `docs/` に正本として持ち、適用開始日で版管理する | 必須 | 実装済 |

## EX. 経費登録フロー(将来実装)

| ID | 要件 | 優先度 | 状態 |
|---|---|---|---|
| REQ-EX01 | 証憑(紙スキャン/メール添付/アップロード)を取り込み、抽出値を構造化(確信度つき)する | 必須 | Phase 1 |
| REQ-EX06 | MoneyForward API 接続を非秘密 config(URL/scopes/client_id)+ 秘密 env(client_secret)で解決し、`ac mf config` で表示・検証できる。秘匿情報はコミット・ログに残さない | 必須 | 実装済 |
| REQ-EX02 | 検証ゲートで為替換算・税区分・証憑要件(電帳法 検索3項目)をチェックし、NG をフラグする | 必須 | Phase 1 |
| REQ-EX03 | NG/低信頼のみ Teams で確認し、承認後に MoneyForward へ経費登録する。摘要に相関キーを付与 | 必須 | Phase 1 |
| REQ-EX04 | 相関キーで会計仕訳を突合し `mf_journal_id` を確定、必要に応じ勘定科目・税区分・摘要を補正する | 推奨 | Phase 1 |
| REQ-EX05 | 不可逆操作(送金・資金移動・削除・権限変更・認証情報入力)はエージェントが実行しない | 必須 | 仕様 |

## TX. 税務・会計(将来実装)

| ID | 要件 | 優先度 | 状態 |
|---|---|---|---|
| REQ-TX01 | 消費税は適用開始日時点で有効な課税方式(2割特例 → 簡易課税)に従い、いずれもみなし控除で扱う | 必須 | 仕様 |
| REQ-TX02 | 国外取引は内外判定で対象外(不課税)とし、国内取引のみ税区分を付与する | 必須 | 仕様 |
| REQ-TX03 | 為替は取引日 TTM 換算を原則とし継続適用する(簡便法・レートソースは税理士確認) | 必須 | 仕様 |
| REQ-TX04 | 税務の最終判断は税理士。エージェントは一次チェックと要確認フラグまで | 必須 | 仕様 |
