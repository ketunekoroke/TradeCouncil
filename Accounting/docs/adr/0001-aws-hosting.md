## 背景

会計経理支援システムの運用フローは「**ローカルの Claude Code で改定 → コミット → push**、
**自動処理サーバは pull して最新ポリシーで動作**」。この自動処理サーバを将来 AWS に置く。
本 ADR は決定済みの **方針を記録** するもので、実装(プロビジョニング・IaC)は行わない。

## 決定事項(方針)

### 1. 構成(将来フェーズ)

- リージョン: **ap-northeast-1**(東京)。
- コンピュート: **EC2 1台**(小型 ARM 想定)+ systemd で常駐ワーカー / watchdog。
- ストレージ: **EBS gp3**(処理状態・SQLite 等)。証憑そのものは MoneyForward / クラウドBox 側に保存。
- ログ: アプリは **構造化 JSON を stdout** に出し、**CloudWatch Logs** へ集約。
- シークレット: client secret 等は **環境変数 / AWS Secrets Manager**。リポジトリに鍵を置かない。

### 2. Git は一方向

- **開発 push → 本番 pull のみ**。EC2 に push 権限を持たせない(常に remote を反映)。
- 例: `*/5 * * * * git -C /opt/accounting pull --ff-only --quiet origin main`。
- `.env` はサーバ固有で Git 管理外。過去取引は取引日基準で旧ルールを維持(再計算しない)。

### 3. 秘匿情報・コンプライアンス

- 証憑の中身(口座番号・カード番号)を **ログに残さない**(CloudWatch にも流さない)。
- 海外アクセス経路の保護(VPN/IP 制限)。**不可逆操作は人間** に限定する(送金・資金移動・削除・権限変更)。
- 税務の最終判断は税理士。サーバ上の自動処理も一次チェック(フラグ)まで。

## 却下した代替案

| 代替案 | 却下理由 |
|---|---|
| 常時稼働のマネージドコンテナ(ECS/Fargate)を最初から | Phase 0/1 の規模に対し過剰。まず EC2 1台で十分(TradeCouncil ADR-0006 と整合) |
| サーバ側でもポリシーを編集可能にする | 一方向 pull の単純さを崩す。改定はローカル → push に一本化(改ざん面の縮小) |
| 証憑を S3 に独自保存 | 電帳法保存は MoneyForward / クラウドBox が担う。二重保存は責務の重複 |

## 影響・注記

- 本 ADR は方針の記録。実プロビジョニング・IaC(Terraform/CloudFormation)は **BL-AC-103** として別途計画する。
- `pyproject.toml` の `aws` extra(boto3 / watchtower)は将来導入用。本体依存には含めない。
