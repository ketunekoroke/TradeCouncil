# 注意点・落とし穴

- **経過措置(80%→50%)は本件に無関係**。2割特例 / 簡易課税ではみなし控除のため、
  誤って経過措置率を適用しない。
- **インボイス6要件の厳格判定は控除目的では不要**(`strict_invoice_mode` OFF)。
  登録番号はあれば取得・記録だけに留める。
- **API の可用性は要検証**: スモールビジネスプランで開発者向け API(経費の API 連携)が有効化できるかを
  実アカウントで確認する(spike の最初のステップ)。
- **クラウドBox API はトライアル提供**。利用条件が将来変わる可能性。本番依存に置かない。
  手動アップロード経路をフォールバックに用意する。
- **expense-api-doc リポジトリは 2026-05-22 にアーカイブ済み**。正確なエンドポイント/パラメータは
  製品ドメインの Swagger(例: expense.moneyforward.com/api)で確認する。
- **摘要の文字数上限** と相関キー埋め込みの可否、**会計 API の摘要フィルタ可否** は要確認
  (不可ならクライアント側で突合)。
- **為替レートソースの税務上の妥当性** は税理士確認(継続適用が前提)。
- **CLAUDE.md とポリシーは利言(強制ではない)**。絶対ルールは pre-commit / CI / PreToolUse フックで
  強制する(出典: code.claude.com/docs/en/memory)。
- **秘匿情報をコミットしない**。ログに証憑の中身(口座番号・カード番号等)を残さない。
- **不可逆操作はエージェントにさせない**(送金・資金移動・削除・権限変更・認証情報入力)。
