# 遵守チェックリスト(タイミング別)

> 強制レベル: [自動]=スクリプト, [CI]=GitHub Actions, [Claude]=Claude Code レビュー, [人]=人手,
> [フック]=pre-commit / PreToolUse

## 1. 抽出時 [自動 / Claude]

- ☐ 必須項目(日付・金額・通貨・取引先・取引内容)を持っている
- ☐ 通貨を特定できている(複数通貨対応)
- ☐ 確信度の低い項目にフラグ

## 2. 経費登録前(検証ゲート)[自動]

- ☐ 為替: 取引日レートで円換算済み(企業は前営業日)。JPY 併記との乖離なし
- ☐ 税区分: 内外判定済み(国外=対象外)、国内は税区分付与済み
- ☐ 証憑: 電帳法検索 3 項目を満たす。基本項目あり。登録番号は取得・記録(あれば)
- ☐ 摘要に相関キーを付与

## 3. 会計連携後・仕訳調整時 [自動 / Claude]

- ☐ 相関キーで会計仕訳を特定し `mf_journal_id` を記録
- ☐ 勘定科目・税区分が会計ポリシーに整合
- ☐ 調整内容を台帳に記録(監査証跡)

## 4. コミット時 [フック]

- ☐ 秘匿情報スキャン(鍵・トークン・.env・口座/カード番号が含まれない)
- ☐ ポリシー文書 lint(適用開始日あり・相互矛盾なし)
- ☐ `check_compliance.py` をパス

## 5. GitHub push / PR 時 [CI]

- ☐ 上記フックと同等のチェックを再実行
- ☐ サンプル取引の検証ゲート通過
- ☐ 失敗ならマージ不可

## 6. 月次レビュー [Claude / 人]

- ☐ フラグ(税理士確認・要確認)の棚卸し
- ☐ サンプル監査(科目・税区分・為替・重複)
- ☐ ポリシーと実処理の乖離を報告

## 7. 四半期 / 決算 [人 / 税理士]

- ☐ 評価替え・要確認事項の整理
- ☐ 為替方針・課税方式の妥当性確認
- ☐ 2割特例 → 簡易課税の移行タイミング確認

## 危険操作の遮断(常時)[フック]

- ☐ 送金・資金移動・削除・権限変更・認証情報入力は PreToolUse フックでブロック
