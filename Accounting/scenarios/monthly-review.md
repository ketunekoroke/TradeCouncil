# シナリオ: 月次レビュー / 経費登録合議(monthly-review)

> Phase 0 = 雛形。手順の骨格のみ。API への書き込み・不可逆操作は自動実行しない(人間が確定)。
> 一次資料: [../docs/manual.md](../docs/manual.md)(日次フロー)・[../docs/accounting-policy.md](../docs/accounting-policy.md)。

ファシリテーター(あなた)は進行と人格の召喚を行い、自分で抽出値・判断の中身を書かない。
人格は [`../.claude/agents/`](../.claude/agents/)(`accountant` / `tax-reviewer`)。

## 入出力

- 入力: `workspace/input/`(証憑・明細)。SharePoint 連携時はシナリオ開始時に
  `python ../shared/sharepoint.py sync --project .`。
- 出力: `workspace/reviews/`(レビュー結果・要確認一覧)。提示時はローカルパスを示す。

## 手順

1. **取り込み**: `workspace/input/` の証憑を文脈として整理する(重複検知。秘匿情報はログに残さない)。
2. **抽出**(`accountant` 召喚): 各証憑から日付・金額・通貨・取引先・取引内容を構造化(確信度つき)。
   為替が絡むものは **内外判定を最初に**(国外=対象外/不課税)。証憑を全人格に等しく渡す。
3. **検証ゲート**: 為替換算(取引日 TTM・継続適用)・税区分・証憑の電帳法 検索3項目をチェック。
   NG/低信頼をフラグ(将来 `scripts/check_compliance.py`)。
4. **税務レビュー**(`tax-reviewer` 召喚): 経過措置の誤適用・strict_invoice_mode 前提・会社特有論点
   (代表者海外在住・海外 SaaS・適格請求書発行=売上側)を確認。要確認は「要確認:税理士」でフラグ。
5. **Teams 確認**: NG/低信頼のみカードで提示 → 人間が修正/承認(自動実行しない)。
6. **経費登録**(承認後・将来 `core/register`): MoneyForward へ登録、摘要に相関キー。電帳法保存は MF が提供。
7. **会計連携の確認**(将来 `core/reconcile`): 相関キーで会計仕訳を突合 → `mf_journal_id` 確定。
8. **仕訳調整**: 必要に応じ勘定科目・税区分・摘要を補正。**調整内容を台帳に記録(監査証跡)**。
9. **月次まとめ**: フラグの棚卸し、ポリシーと実処理の乖離を報告。決算前事項は税理士連携へ。

## 守ること

- どの人格がどの backend/model で動いたかを成果物冒頭に明記(再現性)。
- 反対意見・要確認・少数意見を成果物に残す。穏当に丸めない。捏造しない。
- **不可逆操作(送金・資金移動・削除・権限変更・認証情報入力)は実行しない**。最終確定は人間。
