# ガバナンス・意思決定コア(共通参照資料)

| 項目 | 内容 |
|---|---|
| 位置づけ | **共通参照資料**。TradeCouncil で実装・実証済みのガバナンス・意思決定システムを、ドメイン非依存の「コア」として抽出した解説。docs/ の一次仕様ではなく、本体の仕様・ポリシー・挙動を一切変更しない |
| 効力 | 非規範。規範は本体の一次資料(`docs/02_基本設計書.md` §1.5、`docs/03_運営規程・第0回アジェンダ.md`)にある |
| 性格 | 他のアイデア(流用構想)の土台。これ自体は「実装するもの」ではなく「説明し、磨き込むもの」 |

## 1. 一言で

**「AI が提案し、多視点で審議し、人間が決裁し、決裁済みルールだけが実行を縛る」**
運用フレームワークの中核(コア)の解説。取引(TradeCouncil=実装済み)にも
営業支援(SalesCouncil=構想)にも、同じ構造がそのまま通る — その「同じ構造」の部分だけを
取り出して説明する。

## 2. 文書構成

| 文書 | 内容 |
|---|---|
| [01_ガバナンス・意思決定コア](01_ガバナンス・意思決定コア.md) | 解説本体(二層構成)。**第I部** = 非技術者向け: 考え方・6要素・全体像・実例対応・限界 / **第II部** = 設計者向け: レジストリ・ゲート・会議プロトコル・人格・根拠連鎖・委任・移植手順 |

読み方: 経営向けの説明なら第I部だけで完結する。新しいドメインへ流用を検討するときは
第II部 §12(移植手順チェックリスト)から逆引きするのが速い。
